MyMeds&Me's Technical Test

We've built a simple websocket service that delivers medical data in JSON format.

The data looks like this:

{
  "medications": [
    {
      "antianginal": [
        {
          "name": "nitroglycerin",
          "strength": "0.4 mg Sublingual Tab",
          "dose": "1 tab",
          "route": "SL",
          "sig": "q15min PRN",
          "pillCount": "#30",
          "refills": "Refill 1"
        }
      ]
    }
  ],
  "labs": [
    {
      "name": "Arterial Blood Gas",
      "time": "Today",
      "location": "Main Hospital Lab"
    },
    {
      "name": "BMP",
      "time": "Today",
      "location": "Primary Care Clinic"
    },
    {
      "name": "BNP",
      "time": "3 Weeks",
      "location": "Primary Care Clinic"
    }
  ]
}

Please build a web application that displays all the data.

The endpoint you should connect to is

Once you'll be connected to our service, it will send updates in an interval that can range between 3 and 30 seconds.

Sometimes you'll get duplicate data; in this case you should display it only once. This is the only requirement. Apart from this, feel free to display all the data you get in a way that you think is reasonable.

The service may also return invalid data (i.e. bad formatted JSON), so make sure your application can handle potential issues like this one.

You're free to use any frameworks or libraries that will help you with the challenge, but please be prepared to explain your choices. Also bear in mind that we're more interested in the functionality rather than the appearances; however, you'll get extra kudos if the app looks amazing.

When the project has been completed it should be presented in a state which is maintainable by another developer. To that end:

  • Tell us how to run the app;
  • Tell us how to run your tests;
  • Add comments in any code that needs them.

Good Luck!
